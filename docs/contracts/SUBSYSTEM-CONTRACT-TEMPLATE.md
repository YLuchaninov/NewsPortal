# Шаблон контракта подсистемы

Используй этот шаблон, когда подсистема требует более глубокого durable contract, чем компактный runtime core может удерживать без drift.

## Назначение

Что эта подсистема делает.

## Почему нужен отдельный contract doc

Почему truth этой подсистемы уже нельзя надежно держать только в compact core.

## In scope

Что именно регулирует этот документ.

## Out of scope

Что этот документ намеренно не регулирует.

## External contract

Входы, выходы, API shape, user-visible behavior или integration contract.

## Internal responsibility boundaries

Какие внутренние границы должны оставаться чистыми.

## Data and state rules

Source of truth, derived artifacts, mutation points, persistence и migration notes.

## Runtime and delivery considerations

Startup, environment, platform, packaging, deployment или execution constraints.

## Failure modes

Типичные пути drift или отказа.

## Minimum proof expectations

Какая минимальная proof depth требуется, когда эта подсистема меняется.

## Related files

Связанные code paths, tests, scripts или docs.

## Update triggers

Какие изменения требуют обновить этот contract doc.
